# 📦 wjk — 个人知识管理工具

基于 **Cloudflare Workers** + **Supabase** 构建的个人效率工具，前后端一体，部署即可用。支持备忘、网址收藏、文件备份记录，数据云端同步。

> 🌐 **访问 Worker 地址即可直接使用**，无需单独部署前端。

---

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 📝 备忘 | 新增 / 删除带标签的笔记，支持多行内容 |
| 🔖 收藏 | 保存网址链接，支持标签分类，可直接跳转 |
| 💾 备份 | 选择本地文件一键下载到本地，并在云端记录备份历史 |

- ⚡ 运行在 Cloudflare 全球边缘节点，访问极快
- 🎨 深色 UI，响应式布局，开箱即用
- 🔄 乐观更新，操作立即响应，后台同步数据
- 🆓 Cloudflare Workers 免费额度 + Supabase 免费层即可运行

## 🏗️ 项目架构

```
浏览器
  │
  ▼
Cloudflare Worker（worker.js）
  ├── GET /          → 返回内嵌的前端 HTML 页面
  ├── GET /api/*     → 代理请求到 Supabase REST API
  ├── POST /api/*    → 写入数据到 Supabase
  └── DELETE /api/*  → 删除 Supabase 中的数据
          │
          ▼
     Supabase（PostgreSQL）
       ├── notes（备忘）
       ├── bookmarks（收藏）
       └── files（备份记录）
```

## 🗄️ 数据库结构

```sql
-- 备忘表
create table notes (
  id         bigint generated by default as identity primary key,
  title      text not null,
  content    text,
  tags       text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 收藏表
create table bookmarks (
  id         bigint generated by default as identity primary key,
  title      text not null,
  url        text not null,
  tags       text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 备份记录表
create table files (
  id   bigint generated by default as identity primary key,
  name text not null,
  size bigint,
  date timestamp with time zone default timezone('utc'::text, now()) not null
);
```

## 🚀 部署流程

### 第一步：创建 Supabase 数据库

1. 前往 [supabase.com](https://supabase.com) 注册并新建项目
2. 进入项目 → 左侧菜单 **SQL Editor** → **New query**
3. 将 [`New query`](./New%20query) 文件中的全部 SQL 粘贴进去，点击 **Run** 执行
4. 三张表创建完成

---

### 第二步：获取 Supabase 配置信息

进入项目 → **Project Settings** → **API**，复制以下两项：

| 字段 | 位置 |
|------|------|
| `SUPABASE_URL` | Project URL |
| `SUPABASE_KEY` | Project API Keys → `anon` / `public` |

---

### 第三步：部署 Worker

#### 方式一：Cloudflare 控制台（推荐新手）

1. 打开 [dash.cloudflare.com](https://dash.cloudflare.com) → **Workers & Pages** → **Create** → **Create Worker**
2. 将 [`worker.js`](./worker.js) 的内容全部粘贴进编辑器
3. 点击 **Deploy**

#### 方式二：Wrangler CLI（推荐开发者）

```bash
# 安装 Wrangler
npm install -g wrangler

# 登录 Cloudflare
wrangler login

# 克隆项目
git clone https://github.com/lalal1711/wjk.git
cd wjk

# 部署
wrangler deploy worker.js
```

---

### 第四步：配置环境变量

在 Cloudflare Worker 控制台中：

**Worker 详情页** → **Settings** → **Variables and Secrets** → 添加以下两个变量：

| 变量名 | 值 | 类型 |
|--------|-----|------|
| `SUPABASE_URL` | `https://你的地址.supabase.co` | Plain text |
| `SUPABASE_KEY` | `eyJ...`（anon key） | Plain text |

保存后 Worker 自动重新部署，**访问 Worker 地址即可使用**。

## 📡 API 接口

Worker 同时提供前端页面和 REST API，接口如下：

### 备忘 `/api/notes`

| 方法 | 路径 | 说明 | 请求体 |
|------|------|------|--------|
| `GET` | `/api/notes` | 获取所有备忘（按时间倒序） | — |
| `POST` | `/api/notes` | 新增备忘 | `{ title, content, tags }` |
| `DELETE` | `/api/notes/:id` | 删除指定备忘 | — |

### 收藏 `/api/bookmarks`

| 方法 | 路径 | 说明 | 请求体 |
|------|------|------|--------|
| `GET` | `/api/bookmarks` | 获取所有收藏（按时间倒序） | — |
| `POST` | `/api/bookmarks` | 新增收藏 | `{ title, url, tags }` |
| `DELETE` | `/api/bookmarks/:id` | 删除指定收藏 | — |

### 备份记录 `/api/files`

| 方法 | 路径 | 说明 | 请求体 |
|------|------|------|--------|
| `GET` | `/api/files` | 获取所有备份记录（按时间倒序） | — |
| `POST` | `/api/files` | 新增备份记录 | `{ name, size }` |
| `DELETE` | `/api/files/:id` | 删除指定记录 | — |

> **注意：** 备份功能会将文件**下载到本地**，`/api/files` 仅记录文件名和大小，不存储文件内容。

## ⚠️ 安全说明

当前 RLS 策略为**完全开放**（`Allow all`），任何人只要知道你的 Worker 地址就可以读写数据。

如果仅个人使用，建议在 Worker 中加一层简单鉴权：

```js
// 在 fetch() 开头加入
const AUTH_TOKEN = env.AUTH_TOKEN;
if (path.startsWith('/api') && request.headers.get('X-Token') !== AUTH_TOKEN) {
  return new Response('Unauthorized', { status: 401 });
}
```

然后在 Cloudflare 环境变量中添加 `AUTH_TOKEN`，前端请求时带上该 Header 即可。
